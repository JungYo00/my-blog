## 기존 프로젝트 컨텍스트 (반드시 읽고 반영할 것)
- 이 저장소는 마크다운 → 정적 블로그 사이트를 만드는 SSG 프로젝트다. 프레임워크/번들러/npm 의존성 없이 순수 HTML/CSS/JS + Node 내장 모듈로만 되어 있다 (`package.json`에 `dependencies` 없음).
- `build.js`가 빌드 오케스트레이터다: `dist/`를 통째로 지우고(`fs.rmSync(DIST_DIR, {recursive:true, force:true})`) `posts/*.md`를 읽어 `dist/`에 정적 HTML을 생성한 뒤, `css/`와 `js/`를 `fs.cpSync`로 `dist/css`, `dist/js`에 복사한다.
- `dist/`는 `.gitignore` 처리되어 있고 GitHub Actions(`.github/workflows/deploy.yml`)가 push마다 `node build.js`를 실행해 `dist/`를 GitHub Pages로 배포한다. 즉 **`dist/` 안에 직접 파일을 넣는 방식은 안 된다 — build.js가 실행될 때마다 지워지기 때문**이다.
- 따라서 2048 게임 소스는 저장소 최상위의 별도 소스 디렉토리(예: `games/2048/` — 확정은 네가 spec에서 제안)에 두고, `build.js`가 그 디렉토리를 `dist/`의 적절한 하위 경로로 복사하도록 빌드 스크립트 수정이 필요하다는 점을 spec에 명시해라.
- 기존 코드 스타일: 외부 라이브러리/CDN 없음, 바닐라 HTML/CSS/JS, `css/style.css`에 라이트/다크 CSS 커스텀 프로퍼티 토큰(`--color-bg`, `--color-text` 등)이 이미 정의돼 있고 다크모드는 `[data-theme]` 속성 + `prefers-color-scheme`으로 전환된다 (`js/theme.js` 참고). 새 기능도 이 다크모드 체계와 일관되게 스타일링하는 것을 권장한다.

## spec.md에 반드시 포함할 내용
1. **개요**: 기능 한 줄 요약, 배치 위치(URL 경로 제안, 예: `/2048/`), 블로그 메인 페이지와의 연결 여부(예: 헤더 등에 링크 추가할지 — 네가 제안하고 "결정 필요" 항목으로 표시)
2. **게임 규칙 (2048 표준 규칙)**:
   - 4x4 격자, 시작 시 타일 2개 랜덤 배치(값 2 또는 4, 90%/10% 확률 등 표준 관례 명시)
   - 방향키(↑↓←→)로 모든 타일을 해당 방향 끝까지 밀고, 같은 값의 타일이 부딪히면 합쳐져 2배 값이 되며 점수에 합쳐진 값만큼 가산
   - 한 번의 이동에서 이미 합쳐진 타일은 같은 이동 내에서 다시 합쳐지지 않음
   - 이동 후 빈 칸에 새 타일 1개 랜덤 생성(이동으로 격자에 변화가 있었을 때만)
   - 2048 타일 생성 시 승리 처리(계속하기 옵션 포함할지 결정)
   - 더 이상 이동/병합이 불가능하면 게임 오버
   - 새 게임(재시작) 버튼
3. **점수판 UI**: 현재 점수, 최고 기록(로컬스토리지에 영속 저장) 표시 위치와 스타일
4. **화면/컴포넌트 구성**: 이 기능이 "화면 1개"인지 여러 화면(예: 게임 화면 + 결과 오버레이)으로 나눌 정도인지 판단해서 명시 (CLAUDE.md 규칙: 화면 3개 이상이면 화면별로 Work 서브에이전트를 나눠야 함 — 이 게임은 아마 화면 1~2개 수준일 것으로 예상되니 네 판단을 적어라)
5. **파일 구조 제안**: 새로 만들 파일 목록과 각 파일의 책임, `build.js`에서 수정이 필요한 부분(어떤 함수/단계에 복사 로직을 추가할지)
6. **키보드/접근성**: 방향키 preventDefault(페이지 스크롤 방지), 터치 환경 대응 여부(모바일에서 스와이프 지원할지 — 제안하고 "선택 사항"으로 표시)
7. **테스트 관점**: Review 단계에서 검증해야 할 시나리오 목록 초안(정상 이동/병합, 게임 오버 판정, 승리 판정, 점수 계산, 새로고침 후 최고 기록 유지, 모바일 반응형, 다크모드)
8. **미해결 질문**: 네가 판단하기 애매하거나 사용자 확인이 필요한 항목들을 명시적으로 나열 (예: 게임 오버 후 동작, 2048 도달 후 계속 진행 여부, 블로그 내비게이션 연결 여부 등)

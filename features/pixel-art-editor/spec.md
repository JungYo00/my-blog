# 픽셀 아트 에디터 — spec.md

## 1. 개요

- **한 줄 요약**: 16x16 격자에 마우스/터치로 도트를 찍어 그림을 그리고, 완성한 그림을 PNG 파일로 다운로드할 수 있는 정적 픽셀 아트 에디터 페이지.
- **배치 위치(URL)**: `dist/pixel-art/index.html` → 배포 후 `/pixel-art/` 경로로 접근.
  - 저장소 최상위에 `games/2048/`과 형제 레벨로 `pixel-art/` 소스 디렉토리를 새로 만든다 (게임이 아니므로 `games/` 하위에 넣지 않음).
- **블로그 헤더 링크 추가 여부**: `templates/layout.html`의 `.site-header`에 `2048` 내비 링크 옆에 `<a class="nav-link" href="{{BASE}}pixel-art/index.html">Pixel Art</a>`를 추가하는 것을 제안한다. **(결정 필요)** — 헤더에 링크가 계속 늘어나면 좁은 화면에서 내비게이션이 붐빌 수 있으므로, 추가할지 / 어떤 문구로 넣을지 사용자 확인 필요.

## 2. 에디터 기능 상세

- **격자**: 16x16 도트 그리드. 화면 표시 시 도트 한 칸 = 24px (16 × 24 = 384px 캔버스). 24px는 손가락 터치로도 정확히 찍기 쉬운 크기이면서 화면(특히 모바일 375px 폭)에도 무리 없이 들어가는 크기로 제안.
  - 화면이 더 작은 기기 대응: `@media (max-width: 480px)` 구간에서 18px로 축소(16 × 18 = 288px) 제안.
- **드래그로 연속 칠하기**: 표준 픽셀 에디터 UX이므로 포함할 것을 제안한다. `mousedown` 시 칠하기 시작, `mousemove` 중 눌려있으면 지나가는 칸을 계속 칠함, `mouseup`/`mouseleave`에서 종료. **(결정 필요 — 표준 UX이나 명시적 확인 요청)**
- **지우개 기능**: 별도 "지우개" 버튼으로 모드 전환하는 방식을 제안 (우클릭 방식은 모바일에서 사용 불가하고 컨텍스트 메뉴와 충돌하므로 배제). 지우개는 해당 도트를 배경색(투명 처리를 위해 실제로는 canvas의 `clearRect`로 투명 처리)으로 되돌림. PNG 저장 시 투명 배경이 유지되도록 함.
- **전체 지우기(Clear) 버튼**: 확인 없이 즉시 전체 캔버스를 투명하게 초기화. (되돌리기 불가 — 별도 확인 다이얼로그는 넣지 않는 것으로 제안, 필요시 `confirm()` 추가 가능)
- **기술적 접근 방식 — canvas vs DOM div**:
  - **canvas 기반을 채택**한다. 이유: (1) `canvas.toDataURL('image/png')`로 PNG 저장이 직접적으로 가능해 별도 변환 로직이 불필요, (2) 16x16 = 256개의 DOM 엘리먼트를 만드는 것보다 단순하고 가벼움, (3) 이 프로젝트는 프레임워크 없이 순수 JS로 작성되므로 DOM 조작보다 canvas 픽셀 조작이 코드량이 적음.
  - 구현 방식: 실제 데이터는 16x16 크기의 논리적 픽셀 배열(또는 오프스크린 캔버스)로 관리하고, 화면에는 `imageSmoothingEnabled = false`로 확대해서 그린다. 클릭/드래그 좌표는 화면 캔버스 좌표를 24px(또는 반응형 크기)로 나눠 16x16 논리 좌표로 환산한다.

## 3. 색상 팔레트

- **기본 팔레트**: 16색 제안 (흑백 계열 2~3, 원색 6, 파스텔/보조색 7~8 정도로 구성).
  - 예시 구성: `#000000`(검정), `#ffffff`(흰색), `#808080`(회색), `#ff0000`(빨강), `#ff8c00`(주황), `#ffd700`(노랑), `#00ff00`(초록), `#00bfff`(하늘색), `#0000ff`(파랑), `#8a2be2`(보라), `#ff69b4`(핑크), `#a0522d`(갈색), `#ffb6c1`(연분홍), `#98fb98`(연두), `#87ceeb`(연하늘), `#f5deb3`(베이지)
  - 정확한 16색 목록은 Work 단계에서 최종 확정 (디자인 톤 맞춰 조정 가능).
- **커스텀 색상 선택**: `<input type="color">`를 팔레트 마지막에 추가해 기본 16색 외 임의 색상 선택 가능하게 하는 것을 제안.
- **선택된 색상 표시**: 선택된 팔레트 스와치에 `--color-link` 톤의 테두리(예: `outline: 2px solid var(--color-link); outline-offset: 2px;`)를 주어 강조. 다크모드에서도 `--color-link` 토큰이 자동으로 대비되는 색으로 바뀌므로 별도 다크모드 처리가 필요 없음.

## 4. PNG 저장

- **저장 트리거**: "저장" 버튼 클릭 → 오프스크린(논리) 16x16 캔버스를 기준으로 업스케일된 별도 캔버스에 `imageSmoothingEnabled = false`로 확대해서 그린 뒤, `canvas.toDataURL('image/png')` 결과를 `<a download>` 요소에 설정하고 프로그래매틱하게 클릭해 다운로드를 트리거한다.
- **저장 해상도**: 16x16 원본 그대로 저장하면 실사용성이 낮으므로(대부분 뷰어에서 너무 작게 보임), **16배 업스케일한 256x256으로 저장하는 것을 기본값으로 제안**. **(결정 필요)** — 배율을 16배(256x256)로 고정할지, 사용자가 배율을 고를 수 있게 할지는 확인 필요. 업스케일 시 흐려지지 않도록 저장용 임시 캔버스에도 반드시 `ctx.imageSmoothingEnabled = false`를 설정.
- **파일명**: `pixel-art.png` 고정 이름을 기본으로 제안하되, 매번 같은 이름으로 덮어써지는 것을 원치 않을 수 있으므로 타임스탬프를 포함한 `pixel-art-{YYYYMMDDHHmmss}.png` 형태도 대안으로 제시. **(결정 필요)**

## 5. 화면/컴포넌트 구성

- 이 기능은 **화면 1개**로 충분하다고 판단한다 (에디터 툴바 + 격자 + 팔레트 + 저장 버튼이 모두 한 페이지 안에 들어감, 2048 게임 페이지와 동일한 단일 화면 구조).
- CLAUDE.md 규칙상 "화면이 3개 이상"일 때만 Work 단계를 화면별로 나누므로, 이번 기능은 Work 서브에이전트를 단일 화면 기준으로 1개만 배정해도 충분할 것으로 예상. 다만 필요시 "HTML/CSS 마크업 담당"과 "JS 로직(그리기/저장) 담당"으로 역할 분리하는 것은 가능.

## 6. 파일 구조 제안

새로 만들 파일:

| 파일 | 책임 |
|---|---|
| `pixel-art/index.html` | 페이지 마크업. `templates/layout.html`과 동일하게 다크모드 FOUC 방지 인라인 스크립트, `.site-header`(사이트 타이틀 + 2048 링크 + pixel-art 링크 + 테마 토글) 복제. `../css/style.css` 공통 스타일 + `style.css`(전용 스타일) 링크. 격자 캔버스, 팔레트 컨테이너, 지우개/전체지우기/저장 버튼 마크업 포함. `../js/theme.js` + `editor.js` 스크립트 로드. |
| `pixel-art/style.css` | 픽셀 에디터 전용 스타일 (격자 캔버스 크기/테두리, 팔레트 스와치 그리드, 선택 강조, 버튼, 반응형 레이아웃). `css/style.css`의 `--color-*`, `--font-*` 커스텀 프로퍼티를 재사용해 다크모드 자동 대응. |
| `pixel-art/editor.js` | 에디터 로직: 16x16 논리 픽셀 데이터 관리, 캔버스에 확대 렌더링, 클릭/드래그(마우스+터치) 이벤트로 그리기, 팔레트 선택, 지우개 모드, 전체 지우기, PNG 업스케일 저장. |

`build.js` 수정 부분 (정확한 위치):

- **15번째 줄 근처** (`const GAMES_SRC = path.join(ROOT, 'games', '2048');` 바로 아래)에 다음 상수 추가:
  ```js
  const PIXEL_ART_SRC = path.join(ROOT, 'pixel-art');
  ```
- **108~110번째 줄 근처** (`if (fs.existsSync(GAMES_SRC)) { ... }` 블록 바로 아래)에 동일 패턴으로 추가:
  ```js
  if (fs.existsSync(PIXEL_ART_SRC)) {
    fs.cpSync(PIXEL_ART_SRC, path.join(DIST_DIR, 'pixel-art'), { recursive: true });
  }
  ```
- (결정 필요 항목 반영 시) `templates/layout.html` 22번째 줄(`2048` nav-link) 다음 줄에 `pixel-art` nav-link 추가.

## 7. 접근성/반응형

- **모바일 터치 지원**: `touchstart`/`touchmove`/`touchend` 이벤트를 마우스 드래그와 동일하게 동작하도록 구현하는 것을 제안 (표준 픽셀 에디터 UX와 일치, 2048 게임도 이미 터치 컨트롤 버튼을 별도로 두고 있어 터치 대응 전례가 있음). 터치 드래그 시 페이지 스크롤이 함께 발생하지 않도록 캔버스 영역에 `touch-action: none` CSS 적용 필요.
- **좁은 화면 레이아웃**: 640px 이하에서는 (기존 `css/style.css`의 `@media (max-width: 640px)` 브레이크포인트와 일관되게) 격자를 상단에, 팔레트를 격자 아래 가로 wrap 그리드로 배치. 버튼(지우개/전체지우기/저장)은 팔레트 아래 가로 나열, 공간 부족 시 줄바꿈.
- 버튼에는 `aria-label` 또는 명확한 텍스트 라벨 부여 (2048 페이지의 `aria-label="위로 이동"` 패턴과 일관되게).

## 8. 테스트 관점 (Review 단계 검증 시나리오 초안)

1. 격자 칸을 클릭하면 선택된 색으로 도트가 찍히는지
2. 마우스를 누른 채 드래그하면 지나가는 칸들이 연속으로 칠해지는지
3. 팔레트에서 다른 색을 선택하면 선택 표시(테두리)가 이동하고, 이후 그리기에 새 색이 적용되는지
4. 커스텀 색상(`<input type="color">`)으로 고른 색이 정상적으로 그려지는지
5. 지우개 모드로 전환 후 칠하면 해당 칸이 투명하게 지워지는지
6. "전체 지우기" 버튼 클릭 시 격자 전체가 초기화되는지
7. "저장" 버튼 클릭 시 PNG 파일이 다운로드되고, 실제로 열어보면 그린 그림이 (업스케일된 해상도로) 깨지지 않고 선명하게(흐려짐 없이) 저장되어 있는지
8. 모바일/터치 기기(또는 브라우저 devtools 터치 에뮬레이션)에서 터치로 그리기가 동작하는지, 스크롤이 함께 발생하지 않는지
9. 다크모드 전환 시 페이지 배경/텍스트/버튼 색상이 자연스럽게 바뀌는지 (격자 자체 색상은 사용자가 그린 색 그대로 유지되어야 함)
10. 기존 블로그 홈, 포스트 페이지, 2048 게임 페이지가 이번 변경으로 회귀 없이 정상 동작하는지 (`build.js` 수정이 기존 빌드 로직에 영향 없는지 포함)

## 9. 결정 사항 (사용자 확정, 2026-07-10)

1. **헤더 링크**: 추가한다. `.site-header`에 "2048" 링크 옆에 "Pixel Art" 링크 추가.
2. **PNG 저장 해상도**: 256x256(16배 업스케일) 고정. 배율 선택 UI는 넣지 않는다.
3. **저장 파일명**: `pixel-art.png` 고정 이름 (브라우저가 중복 시 자동으로 `(1)`, `(2)` 등을 붙임).
4. **드래그 연속 칠하기**: 포함한다.
5. **전체 지우기 확인 다이얼로그**: 넣지 않는다 — 클릭 즉시 실행 (기본 제안 그대로 채택).
6. **기본 팔레트 16색**: 섹션 3에 제시된 예시 목록(`#000000`, `#ffffff`, `#808080`, `#ff0000`, `#ff8c00`, `#ffd700`, `#00ff00`, `#00bfff`, `#0000ff`, `#8a2be2`, `#ff69b4`, `#a0522d`, `#ffb6c1`, `#98fb98`, `#87ceeb`, `#f5deb3`)를 그대로 최종 확정한다.
